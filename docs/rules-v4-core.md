## 0. 通貨と盤面の基本

### 工数キューブ(唯一の通貨)

- **タスクカード**には以下が刷ってある:
  **埋めるスロット(§2)/ 系統(📋dir・🎨des・⚙eng のいずれか1つ)/ 必要工数 / 上限Lv / 疲労値(1 または 2)/ 実行コスト(予算)/ 列(起点・中盤・仕上げ)**
  - 例:「トップページ実装 → スロット【トップ】/ ⚙4 / Lv2可 / 疲労2 / コスト2 / 中盤」
  - v4.0 のタスクは**単一系統**(混合系統タスクは将来の拡張)
- プレイヤーがタスクに1週座ると、**その系統の自分のスキル値ぶん**のキューブを積む(スキル0の系統には座れない)
- キューブは**タスクに残る**。人は毎週手元に戻り、続けるか離れるかを選び直す(拘束なし)
- 複数人で座れば足し算。**同じ人が主担当+残業で同じタスクに2枠使うのも可**(残業してやり切る、は自然な行動)

### 納品と品質

- キューブが必要工数に達したタスクは、**週末に納品できる**(チームで選択):
  - **そのまま納品** → スロットが **Lv1** で埋まる
  - **積み増し続行** → さらに +`qualityOvershoot`(暫定2)個積んでから納品すると **Lv2**(上限Lv1 のタスクは不可)。
    必要工数+overshoot を超えては積めない
- 納品時に**実行コスト(予算)を支払う**。予算が足りなければ納品できない(積んだキューブは残る)
- **改修**:納品済みの Lv1 スロットには「改修」として座れる(系統=スロットの系統)。
  +`upgradeCost`(暫定3)個積むと Lv2 に上がる。**どのタスク由来の Lv1 でも同じ**
  (納品前に丁寧にやれば+2、納品後に直すと+3。**安物買いの銭失い=技術的負債の学び**)

### 🔥(炎上)

- 🔥1個 = そのタスクの必要工数 +1(スコープクリープ)。**未納品のタスクにのみ乗る**。納品すると🔥は消える
- 炎上カードの対象は**名指しではなく条件式**(例:「キューブ最多の進行中タスク」「仕上げ列の未納品タスク」。
  同数・複数該当は PM 帽子が裁定)。タスクがドラフト制になったため、v2.1 の「タスク名指しデッキ」は廃止
- **延焼**:同一タスクの🔥が `fireOutbreakThreshold`(暫定3)個目に達したら、置く代わりに
  **同じ列の進行中タスク全部に🔥+1、CS-1**
- **大炎上**:進行中の全タスクに🔥+1
- **消火**(週アクション):任意のタスクの🔥1個を除去。**系統・スキル不問**(誰でもできる)。
  積む(スキル値ぶん)より効率は劣るが、延焼・大炎上の**保険**であり、系統が合わない週の受け皿

### 疲労(0〜4 の数値トラック)

- タスク/改修に座った週:+タスクの疲労値(1 or 2)。残業:さらに+1(即時)
- **2以上:残業禁止**。**4到達:限界イベント**(カードを1枚引いて解決し、疲労2に戻る)
- 休憩(週アクション):-2。フェーズ末:全員-1
- ※ v3 の「Lv0〜3」から**上限4に拡張**(シミュレーションで疲労の崖が急峻すぎたため)

## 1. ゲームの流れ

**1ゲーム = 4フェーズ(企画/設計/開発/テスト)、1フェーズ = 3週。**
検収条件・タスク候補はフェーズのテーマに沿って公開される(コンテンツ側で担保)。

### 1-1. スコープ会議(フェーズ開始)

> **原則:タスクの完了は目的ではない。成果が目的。**

1. **検収条件カード**を `acceptancePerPhase`(暫定2)枚公開する。各条件は
   **スロット×要求Lv**を指す(例:「【トップ】が Lv2」)。**条件は累積する**
   (過去フェーズの未約束・未達成の条件も場に残り、後から約束できる)
2. チームで相談し、**約束する条件を選ぶ**(最終決定は PM 帽子)。
   **一度した約束は、達成するか交渉で取り下げるまで続く**(再約束は不要)
3. **タスク候補**を場に補充し(常時 `draftPool`(暫定8)枚)、やると決めたタスクを**レーンに配置**する(= WBS):
   - 列の文法:カードに刷られた列(起点/中盤/仕上げ)にしか置けない。
     中盤・仕上げのカードは、**直前の列**にタスクが1枚以上配置されていることが条件
   - 配置後の**着手・納品の順序は縛らない**(並行作業は現実にもある。列は構造の表現)
   - 同じスロットを埋める手段は複数ある(安い道=工数少・Lv1上限/堅い道=工数多・Lv2可)。
     **やらなくていいタスクが常に存在する**
   - タスクの配置・追加はスコープ会議以外でも可(週の朝会で場から追加してよい。ただし候補の補充は会議時のみ)
4. **見積もりの責任**:約束済み・未達の検収条件は、フェーズ末ごとに **CS -`commitPenaltyCs`**。
   約束していない条件に罰はない(ただし最終検収 §2 には響く)

> 検収条件の総需要はチームの供給より常に大きい。**全部は約束できない。**
> どの成果を約束し、どの手段で行き、何を見切るかがこの会議の本体。

### 1-2. 週(×3)

1. **週初のトラブル**:炎上カード1枚+イベントカード1枚を解決する。イベントには**差し込み**を含む。
   差し込みは**すべて割り込みレーン(列文法の外)にタスクとして置かれ**、通常どおり座って積める:
   - **手戻り**:納品済みスロット1つに手戻りキューブ N 個が乗る。**乗っている間、そのスロットは
     検収上「未達」扱い**。同系統で N 個積めば除去(Lv は元に戻る)
   - **バグ報告**:工数2の対応タスクが出る。**未対応の間、フェーズ末ごとに CS-1**。完了すれば止まる
   - **相談ごと**:工数2の任意タスク。完了すると報酬(予算+2、次の検収条件が易しくなる 等)。放置しても罰なし
2. **朝会(同時配置)**:各自「今週の自分」を決める:
   - **座る**(場のタスク/改修/手戻り対応。自分のスキルぶん積む)
   - **学習**(系統を選びスキル+1。**反映は来週から**。上限 skillMax=3)
   - **休憩**(疲労-2)
   - **消火**(🔥1個除去)
   - 任意で**残業**:+1枠(**座る/消火のみ**。学習・休憩は不可)。即時疲労+1。疲労2以上は不可
3. **週末**:必要工数に達したタスクの納品判定(§0)。納品→スロットが埋まり、達成された検収条件はその場で達成扱い

- **個人能力**(§3)は行動枠を消費しない(フリーアクション。各カードの記載タイミングで、フェーズ1回)

### 1-3. フェーズ終了

1. **約束の清算**:約束済み・未達の検収条件1つにつき CS-`commitPenaltyCs`
2. **必要工数に達したまま未納品のタスク**があれば、納品するか持ち越すかをチームが選ぶ
   (持ち越して積み増し続行=約束は未達のまま清算される。Lv2 を取るために CS を払う、は正当な択)
3. バグ差し込みの未対応ペナルティ(CS-1/件)
4. 疲労の自然回復(全員-1)、次フェーズのスコープ会議へ
5. **最終フェーズ終了時は最終検収**:公開された全検収条件について
   **未達成 = CS-2 / Lv2要求を Lv1 で充足 = CS-1 / 充足 = 0** を清算し、CS ≥ 0 なら勝利
- CS が 0 未満になった時点で即時敗北(従来どおり)

## 2. プロダクトボード(出口。= お客様の成果)

- 作っている Web サイトの**完成図ボード**。スロット(サイトマップ/ワイヤー/デザイン/トップ/下層/CMS…)に、
  納品した成果物が **Lv 付きで**はまっていく。タスクカードの「埋めるスロット」欄が対応を定める
- **検収はすべて検収条件カード経由**(v4.0 では「中間検収」を廃止。フェーズ末は約束の清算のみ、
  ボード全体の評価は最終検収の1回。シミュレーションもこの形でバランス済み)
- **技術的負債の可視化**:Lv1 のスロットは炎上カードの対象条件に含まれやすい(雑に作った所は燃える)※コンテンツ側で表現
- スロットの**完成ボーナス**(例:「スタイルガイド完成:以後デザイン系の必要工数-1」)は v4.1 のコンテンツ設計
  (置き場所を選ぶ楽しさ=報酬レイヤーはここに宿す)

## 3. メンバーと PM 帽子(職業クラス制の廃止)

「PM/デザイナー」という固定クラスをやめ、**能力(メンバーカード)**と**責任(PM 帽子)**を分離する。

### メンバーカード

- 各プレイヤーに1枚(配布 or ドラフト)。**スキルプロファイル+個人能力+経歴フレーバー**を持つ:

| 例 | 📋dir | 🎨des | ⚙eng | 個人能力(フェーズ1回・行動枠を使わない) |
|---|---|---|---|---|
| 元エンジニアの何でも屋 | 1 | 1 | 1 | **マルチタスク**(パッシブ):残業の追加疲労なし |
| デザイン寄り | 1 | 2 | 0 | **磨き込み**:週末に、納品済みスロット1つを Lv1→Lv2 |
| マネジメント寄り | 2 | 1 | 0 | **段取り**:朝会で宣言。今週自分の積むキューブ+1 |
| 開発寄り | 0 | 1 | 2 | **自動化**:未納品タスク1つの必要工数-1(いつでも) |

- 0 の系統には座れない。1 あれば座れる=遅くても手伝える
- 5人プレイはカードを1枚増やすだけ。学習でプロファイルは育つ(上限3)
- ❓ プロファイルのバリエーション数・配布方式はコンテンツ設計で

### PM 帽子(責任=役割)

- **PM マーカー**を1人が被る(セットアップで決定。教育モードではフェーズごとに回す選択ルール)
- 帽子の権限:
  - **スコープ会議の最終決定**(約束・WBS 配置)
  - **交渉**(フェーズ1回・いつでも):①約束1つの清算を1フェーズ猶予 ②約束1つの取り下げ(即時 CS-1、以後の罰なし)
    ③スコープ会議中ならタスク候補2枚の引き直し — のいずれか
  - **追加請求**(フェーズ1回:CS-1 で予算+`extraBillingBudget`)
  - **裁定**:炎上対象の同数タイブレークなど、ルールが指定しない選択
- スキルと責任が独立するので「PM 寄りスキルの人が帽子を被るとは限らない」も起きる(実務どおり)

## 4. そのまま残るもの

- **CS 単一勝敗トラック**(初期値はプロジェクトシート。暫定5。0未満で即敗北)
- **予算**(初期値はプロジェクトシート。暫定18。納品コストで減り、追加請求・相談ごと報酬で増える)
- **疲労・限界イベント**(§0 の数値トラック化のみ変更)
- **個人目標(働き方の個性)・マイルストーン**
- **イベントソーシング・GameConfig 外出し**などアーキテクチャ原則すべて
- ⚠ **クライアント QCD 重みは v4.0 では一旦適用外**(検収・清算はすべて重みなしの整数。
  シミュレーションも重みなしで検証済み。クライアントの個性は「検収条件の内容・差し込みの傾向」で
  表現する方向で §7 で再検討)

## 5. 削除台帳(v3 → v4)

| 削除 | 理由/行き先 |
|---|---|
| 専門席・人手席 | 工数キューブに統合 |
| やっつけ解決+CS債務 | 不要(下手=遅いだけ) |
| 過剰スペック割引 | 不要(上手=速いが既に報酬) |
| 収穫の選択・じっくり(拘束) | 積み増し=品質に統合 |
| 品質閾値の独立判定・中間検収 | 検収条件カード+最終検収に統合 |
| 締切許容数と締切利息の併存 | 「約束未達 = CS-1/フェーズ」1本に統合 |
| カード個別の依存チェーン | レーン文法(直前列に1枚)に置き換え |
| タスク名指しの炎上デッキ | 条件式ターゲット(キューブ最多 等)に置き換え |
| 秘匿要件の受動ドロー | 要件定義アクション(v4.1 で再導入検討) |
| EP トラック | 削除済み(v3) |
| 固定4クラス | メンバーカード(能力)+ PM 帽子(責任) |
| 「タスク全部やる」前提 | 検収条件ベース(タスク=手段) |
| クライアント QCD 重みの乗算 | v4.0 では適用外(§7 で再検討) |

## 6. GameConfig(暫定値)

```

initialBudget: 18
extraBillingBudget: 3
```

## 7. 未決事項

- ❓ クライアントの個性の再接続(QCD重みに代わり「検収条件の構成・差し込みデッキの傾向」で表現する案)
- ❓ 差し込みの「見切り」が稀(シミュで拾った率90%超)。見切る理由の設計余地
- ❓ 品質重視プレイの見返り(v4.1 完成ボーナスが前提。それまで Lv2 は約束・最終検収の要求ベース)
- ❓ 外注:「予算4+CS1 でキューブN個を即積み」の形で v4.0 に入れるか
- ❓ スコープカット報酬(候補を明示的に断ると予算+1)を入れるか → 暫定:入れない(断り=単に取らない、で足りる)
- ❓ 文脈コンボ(同じ人が続けて積むと+1)→ 暫定:入れない
- ❓ プロダクトボードのスロット構成・完成ボーナス(v4.1)
- ❓ メンバーカードの種数・配布方式、PM 帽子ローテーション
- ❓ モードプリセット(追体験/トリアージ)・連続キャンペーン(v4.2)

## 9. 経済シミュレーション所見(2026-07。モンテカルロ、4戦略×500ゲーム×設定スイープ)

- **合格設定**:acceptancePerPhase=2 / interruptChance=0.3 / commitPenaltyCs=1 で混合勝率41.5%。
  手空きアクション率は全設定で0%
- **欲張り(全部約束)は支配戦略にならない**(勝率0%で最下位。罰しすぎの気配は要プレイテスト)
- **育成先行が最強(73.8%)**:「学習=恒久的に速くなる」は意図どおり
- **発見した罠→本稿でルール化済み**:Lv2要求×安い道の約束=返済不能債務 → 改修(+upgradeCost)と
  交渉の取り下げ(CS-1打ち切り)で有限の・教育的な失敗に変換
- **本稿での対応**:疲労の崖 → fatigueMax 4 に拡張。中間検収 → 廃止(シミュのモデルと一致)
- 残る要調整:品質重視の見返り(v4.1)、差し込みの見切り設計。モデルは予算・レーン・個人能力を省略した簡易版
