# ルール v2 提案 — メカニクス借用による「ジレンマ強化」たたき台

> **このファイルは提案(discussion draft)であり、ルールの正ではない。**
> 章ごとに採否・数値を議論し、確定した章から [RULES.md](RULES.md) に反映 → GameConfig フラグ付きで実装する。
>
> 背景:ステージ1のプレイテストで「どこに置くかのジレンマが薄い」という課題が出た。
> 原因は ①トークンが余る ②誰がどこに置いても同じ(代替可能) ③個人目標がチーム行動と衝突しない、の3点。
> 名作ボードゲームのメカニクスを借用して、これらを構造的に解決する。

## 導入ロードマップ(複雑性の予算管理)

一度に全部入れず、1段階ずつプレイテストして採否を決める。

| 段階 | 導入する章 | ねらい |
|---|---|---|
| v2.1 | §1 炎上システム + §5 EP・マイルストーン | 圧力と個人インセンティブ。実装が軽く効果が大きい |
| v2.2 | §2 資格ワーカー + §3 資源と高品質化 | 「代替できない」と「Lv2の作り方」の本命 |

## §1 炎上システム 🔥(借用元:パンデミック)

**狙い**:放置の代償を作る。「全部やればいい」を壊し、消火(守り)と建設(攻め)で
トークンの使い道を競合させる。

### ルール案

1. **炎上デッキ**を新設。各タスクタイルを指す対象カードで構成する
2. フェーズ開始時、既存のイベントカードに加えて**炎上カードを `firePerPhase`(暫定2)枚**引き、
   対象タスクに🔥トークンを1個ずつ置く(解決済みタスクが対象なら何も起きない)
3. 🔥1個につき、そのタスクの**必要行動トークン +1**(スコープクリープ。要件が膨らんでいく)
4. **延焼(アウトブレイク)**:同一タスクに **`fireOutbreakThreshold`(暫定4)個目**の🔥は置けず、
   代わりに依存ツリーの**子タスク全員に🔥1個ずつ** + **CS -1**。子がいなければ同列の各タスクへ
5. **消火**:プランニング中、行動トークン1個で自タスク・自系統の🔥1個を除去(`extinguishCost: 1`)。
   資格外のタスク(§2)は2個必要
6. **大炎上カード**(エピデミック相当)を炎上デッキに `epidemicCount`(暫定2)枚混ぜる:
   引いたら炎上捨て札をリシャッフルして山に戻し、任意のタスク1枚に🔥2個
   (= 一度燃えたタスクはまた燃えやすい)

### 既存ルールへの影響

- §2-1(フェーズ開始)に「炎上カードを引く」を追加
- §2-3(実行)の手順1は「必要トークン数 = 基本値 + 🔥数」で判定
- 未解決タスクの持ち越し時、🔥も持ち越す(燃え続ける)

### GameConfig 追加(暫定値)

```
fireEnabled: true
firePerPhase: 2
fireOutbreakThreshold: 4
fireOutbreakCsPenalty: 1
extinguishCost: 1          // 資格内
extinguishCostOutside: 2   // 資格外(§2導入後)
epidemicCount: 2
```

### 未決事項

- ❓ 解決時に🔥が残っていた場合の追加ペナルティ(疲労+1?)は必要か → 暫定:なし(必要トークン増のみ)
- ❓ 消火は実行ステップ中にもできるべきか → 暫定:プランニング中のみ
- ❓ 大炎上の対象選択は誰が決めるか → 暫定:PM

## §2 資格ワーカー(借用元:ダーウィンズジャーニー + エンデバー深海)

**狙い**:「誰が置くか」に意味を持たせ、代替不能性を作る。学習を「数値+1」から
「人を育てる」に変える。

### ルール案

1. 行動トークンを廃止し、**ワーカー**に置き換える。各プレイヤーはワーカー
   `workersPerPlayer`(暫定4)体を持ち、毎フェーズ全員戻ってくる(補充ではなく回収式)
2. 各ワーカーは**資格マーク**(ディレクション/デザイン/エンジニアリング、各Lv1〜2)を持てる。
   初期状態:ロールの系統 Lv1 の資格を持つワーカー2体 + 無資格2体
3. タスクの席に**資格ゲート**を設ける:
   - **通常席**:無資格でも置ける(人手としてカウント)
   - **専門席**(タイルに0〜2席):指定系統の資格 Lv 以上のワーカーのみ。
     専門席が埋まらないとタスクは解決できない(現在の「スキル条件」の置き換え)
   - **専門席ボーナス**:Lv2 資格ワーカーを Lv1 専門席に置くと実行コスト -1(過剰スペック)
4. **学習タイル** = ワーカー1体に資格を付与/昇格(コスト:ワーカー1体配置 + 現Lv+1 フェーズ…ではなく
   暫定:**同一ワーカーを学習タイルに置くと次フェーズ開始時に資格+1**。育成中はそのワーカーを使えない)
5. プレイヤーの「スキルレベル」(v1)は廃止し、**チームの実力 = ワーカーたちの資格の合計**になる

### 既存ルールへの影響

- §2-2(プランニング):配置は「ワーカー単位」になり、どの個体を置くかが選択になる
- §2-3 手順2(スキル条件)→ 専門席ゲートに置き換え
- §4(疲労):疲労はプレイヤー単位のまま(ワーカー単位にすると煩雑)。
  重タスクの疲労はそのタスクに置いた**ワーカーの持ち主**に入る
- 限界イベントの「次フェーズ補充-1」→「次フェーズ、ワーカー1体が寝込んで使えない」に読み替え
- §5(スキルと成長)は本章で全面置き換え

### GameConfig 追加(暫定値)

```
workerQualificationEnabled: true
workersPerPlayer: 4
initialQualifiedWorkers: 2   // ロール系統Lv1資格つき
trainingTurns: 1             // 学習タイルに置いてから資格が付くまでのフェーズ数
overqualifiedDiscount: 1     // Lv2資格をLv1席に置いた時のコスト割引
```

### 未決事項

- ❓ 1ワーカーが複数系統の資格を持てるか → 暫定:可(ただし学習は1回1系統)
- ❓ 資格Lv2への昇格条件 → 暫定:Lv1資格ワーカーを再度学習タイルへ
- ❓ ワーカー個体の識別(物理版はミープル+リング、デジタル版は ID)

## §3 資源と高品質化(借用元:パンデミックの特効薬)

**狙い**:Lv2 成果物を「重いタスクのおまけ」から「チームで狙って作る中期プロジェクト」へ。
品質戦略がエンジン構築(§4)と接続する。

### ルール案

1. **資源3種**を新設:📋知見(ディレクション)/ 🎨意匠(デザイン)/ ⚙技術(エンジニアリング)。
   チーム共有ではなく**個人の手元**に入る(EP・個人目標と絡むため)
2. 入手手段:
   - タスク解決時の産出(タイル定義を「Lv2成果物」→「資源1〜2個」に差し替え。獲得は参加者で分配、
     端数は PM が割り振る)
   - §4 導入後:稼働タスクの毎フェーズ産出
3. **高品質化(クラフト)**:プランニング中、行動トークン(ワーカー)1個 +
   **同系統の資源 `craftCost`(暫定3)個**を支払うと、その系統の **Lv2 成果物**を獲得する
   (= ウィルス特効薬。5枚同色カードの感覚)
4. Lv1 成果物は従来どおりタスク解決で獲得。**タイル直付けのV2 成果物は廃止**し、本ルートに一本化
5. 品質(Q)判定は変更なし:そのフェーズに獲得した Lv2 成果物数で判定
   → 「今フェーズ中にクラフトが間に合うか」の計画ゲームになる

### 既存ルールへの影響

- §6 タスクタイル定義:`deliverables` の Lv2 を資源産出フィールドに置き換え
- 限界イベント「品質の妥協」:Lv2成果物の劣化 → **資源2個を失う**に変更(劣化対象がない事故を防ぐ)
- 要件カード「ポートフォリオ掲載OK」:ボーナス成果物 → 資源+1 に変更

### GameConfig 追加(暫定値)

```
resourceCraftEnabled: true
craftCost: 3            // 同系統資源でLv2成果物1個
craftActionCost: 1      // 消費する行動トークン(ワーカー)数
mixedCraftCost: 4       // ❓ 異系統混合で作れるなら(暫定:不可)
```

### 未決事項

- ❓ 資源の持ち越し上限(インフレ防止)→ 暫定:上限なしで一度テスト
- ❓ 資源の譲渡(パンデミックの「知識の共有」のような同タスク上での受け渡し)→ 暫定:あり。
  同じタスクに参加したプレイヤー間でのみ譲渡可(これも相談と配置の理由になる)

## §4 タスクドラフトと産出エンジン(借用元:イッツアワンダフルワールド)

**狙い**:「どのタスクを場に出すと良いエンジンになるか」をチームの議論にする。
スコープを自分たちで決める = 納期判定の母数を自分たちで握る。

### ルール案

1. タスクタイルの自動公開を廃止し、**タスクカードの手札**にする:
   フェーズ開始時、各プレイヤーに `taskCardsPerPlayer`(暫定3)枚配る
2. **スコープ会議**(新ステップ。フェーズ開始とプランニングの間):
   全員で相談し、手札から**合計 `tasksPerPhase` 枚まで**を場(依存レーン)に出す
   - カードには**列番号**(起点 / 中盤 / 仕上げ)が刷ってあり、その列にしか置けない
   - 一部のカードは**前列条件**を持つ(例:「中盤列に『設計』タグのタスクがあること」)
3. **スコープカット**:場に出さなかったカードは捨てて、1枚につき**予算+1 または 資源1個**
   (IWW のリサイクル。「断る勇気」に報酬を)
4. **産出(エンジン)**:一部のタスクは解決後に**稼働状態**になり、
   以後毎フェーズ開始時に資源や予算を産出する(例:スタイルガイド → 毎フェーズ🎨1)
   - 稼働タスクは盤面に残り続ける(🔥の延焼対象にもなる = 動いてるシステムは燃える)
5. **納期(D)判定**:母数は「場に出したタスク」のみ。出しすぎは自殺、絞りすぎは品質・産出が痩せる

### 既存ルールへの影響

- §2-1(フェーズ開始)を「配布 → 炎上 → イベント → スコープ会議 → 補充」に再構成
- §7(セットアップ)4 のタスク公開 → 初期手札配布 + 第1スコープ会議に置き換え
- タスクカードに新フィールド:列番号 / 前列条件 / タグ / 産出(稼働時)

### GameConfig 追加(暫定値)

```
taskDraftEnabled: true
taskCardsPerPlayer: 3
scopeCutBudget: 1        // カット1枚あたりの予算回復
maxBoardTasks: 6         // 場に出せる上限
```

### 未決事項

- ❓ 手札は個人持ちか共有か → 暫定:**個人持ち・中身は公開**(相談はできるが、出す/カットの最終決定は持ち主。
  奉行対策と交渉の種)
- ❓ 使わなかった手札の持ち越し → 暫定:持ち越し可、上限5枚
- ❓ タグとプロジェクトカードのシナジー(TM式)はこの章に含めるか → 暫定:含める(カード定義時に設計)

## §5 EP とマイルストーン(借用元:白ブラス + テラフォーミングマーズ)

**狙い**:奉行問題への二段構え。「利他が利己になる」(ブラス)と「公開の早取り競争」(TM)で、
全員が自分の意思で動く理由を作る。

### ルール案

#### EP(Engagement Point)の定義 — RULES.md §9-9 の答え

> **EP = 自分の仕事が他人に使われた回数。**

EP が入る契機(各 +1):

1. 自分が作成者の**成果物**が、他プレイヤーの高品質化・タスク解決条件に使われた
2. 自分が**親タスク**に参加していて、その子タスクを他プレイヤー(自分不参加)が解決した
   (= 自分の下準備の上で他人が成果を出した)
3. 自分の**稼働タスク**(§4)の産出資源を他プレイヤーが使った
4. 自分が**消火**(§1)したタスクを、そのフェーズ中に他プレイヤーが解決した

- EP は公開トラック(隠すと集計が辛い。物理版を考慮)
- 個人目標カードに EP 条件を追加:「EP 5 以上」「EP でチーム1位」など

#### マイルストーン(公開・早取り)

セットアップ時に3枚公開。**最初に達成した1人だけ**が取れる:

| 例 | 条件 |
|---|---|
| 火消し番長 | 累計消火 5 回 |
| 品質職人 | Lv2 成果物の作成 2 個 |
| 育成の人 | 資格付与/昇格 3 回 |
| 縁の下 | EP 4 |
| 鬼の工数 | 1フェーズにワーカー5体以上をタスクに配置 |

#### 個人勝利条件の再定義

> 個人勝利 = **チーム勝利** かつ(**秘匿の個人目標達成** または **マイルストーン獲得**)

- 秘匿目標は配られた2枚から1枚選ぶ方式に変更(自分の戦略に合わせられる)
- チーム敗北時は従来どおり全員敗北(ここは協力ゲームの背骨なので変えない)

### GameConfig 追加(暫定値)

```
epEnabled: true
milestonesEnabled: true
milestoneCount: 3
personalGoalChoices: 2   // 配られる枚数(1枚選ぶ)
```

### 未決事項

- ❓ EP のインフレ調整(契機が多すぎないか)→ プレイテストで契機を間引く前提
- ❓ マイルストーンの同時達成 → 暫定:アクション解決順で先の者
